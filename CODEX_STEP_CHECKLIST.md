# Codex Step Checklist

**목적:** Step 작업을 시작할 때마다 빠뜨리지 않도록 반복 절차를 고정한다.

---

## 필독 사항

### 0. 최신 변경사항 먼저 반영

작업 시작 전에 항상 최신 변경사항을 확인한다.

```bash
git pull https://github.com/skek99110199-bit/Sound-learn.git
```

주의:

- 작업 전 먼저 pull
- 작업 후에는 본인 변경사항도 반드시 업로드

---

### 1. `README.md` 먼저 읽히기

Codex를 새로 켤 때마다 가장 먼저 `README.md`를 기준 문서로 삼는다.

권장 순서:

1. `/home/sunhan/Sound-learn/README.md` 확인
2. `/home/sunhan/Sound-learn/Sound-learn/README.md` 확인
3. 현재 진행할 Step 기준 문서 확인

Codex에게 바로 이렇게 말하면 된다:

```text
README.md를 먼저 읽고 프로젝트 구조와 현재 Step 기준을 이해한 뒤 작업해.
```

---

### 2. Step 시작 전 메타 프롬프팅 먼저 받기

각 Step을 바로 진행하지 말고, 먼저 메타 프롬프트를 뽑는다.

예시:

```text
step1부터 진행할 예정이야. 이 Step을 Codex로 잘 진행하려면 어떤 메타 프롬프트로 시작하면 좋은지 작성해줘.
```

또는:

```text
step2를 진행할 예정이야. README 기준 목표를 반영해서 작업용 메타 프롬프트를 만들어줘.
```

---

### 3. 만든 메타 프롬프트를 실제 작업 프롬프트로 다시 넣기

메타 프롬프트를 받은 뒤 그대로 복붙해서 실제 작업을 시작한다.

반드시 마지막에 아래 문장을 붙인다:

```text
진행할 때에는 plan을 세우고 모르는 부분과 결정하기 어려운 부분을 물어보면서 진행해.
```

최종 예시:

```text
<메타 프롬프트 내용>

진행할 때에는 plan을 세우고 모르는 부분과 결정하기 어려운 부분을 물어보면서 진행해.
```

---

## 추천 작업 순서

1. `git pull`
2. `README.md` 읽히기
3. 진행할 Step 말하기
4. 메타 프롬프트 요청
5. 메타 프롬프트 복붙
6. 마지막 문장 추가
7. 실제 작업 진행

```text
stepN을 진행할 예정이야. README 기준 목표를 반영해서 작업용 메타 프롬프트를 작성해줘.
```

### 템플릿 3. 실제 작업 시작

## 체크리스트

- [ ] 작업 시작 전 `git pull` 했다
- [ ] `README.md`를 먼저 읽혔다
- [ ] 진행할 Step을 명시했다
- [ ] 메타 프롬프트를 먼저 만들었다
- [ ] 메타 프롬프트를 실제 작업 프롬프트로 다시 넣었다
- [ ] 마지막 문장 `plan을 세우고...` 를 붙였다

---

## 한 줄 요약

Step 작업 전에는 항상 **`git pull -> README 학습 -> 메타 프롬프트 작성 -> 실제 프롬프트 실행`** 순서로 진행한다.
